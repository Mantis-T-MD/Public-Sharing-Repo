Copy this entire block into any AI agent's context on the work laptop or Razer Blade:

# Any device on the Tailscale network can communicate with it.

S23_TAILSCALE_IP="100.109.67.85"

# Post a question to the S23 Dashboard (user sees it, responds):
curl -sf -X POST ${DECK_URL}/api/agent/question \
  -H 'Content-Type: application/json' \
  -d '{"text":"Hello from the laptop agent!"}'

# Long-poll for response (blocks up to 120s):
# Replace QID with the "id" from the question response
curl -sf "${DECK_URL}/api/agent/response?id=QID&wait=120"

# Send a one-way message (no response expected):
curl -sf -X POST ${DECK_URL}/api/agent/message \
  -H 'Content-Type: application/json' \
  -d '{"text":"Status: task completed"}'

# Check inbox (messages user sent from Dashboard):
curl -sf "${DECK_URL}/api/agent/inbox?wait=5"

API Reference

Base URL: http://100.109.67.85:8080

POST /api/agent/question

Post a question from an agent. User sees it in the Dashboard and responds.

Request:  {"text": "What should I work on?"}
Response: {"id": "q-5", "ok": true}

# Then long-poll for user's answer:
GET /api/agent/response?id=q-5&wait=120
Response: {"id": "q-5", "text": "Work on the auth module"}

POST /api/agent/message

Send a one-way message to Dashboard (no response expected).

Request:  {"text": "Build completed successfully"}
Response: {"ok": true}

GET /api/agent/inbox?wait=N

Poll for unprompted messages from the user (sent via Agent mode in Dashboard).

Response: {"id": "u-3", "text": "Hey, switch to the auth branch"}

GET /api/events

Server-Sent Events stream. Real-time notifications of all agent activity.

Event types:
  agent_question  — agent posted a question
  agent_message   — agent sent a status message
  goal_update     — goal topology changed

GET /api/goals

Current goal topology (read-only). Useful for agents to understand context.

Shell Script (deck-remote.sh)

Self-contained script — copy to any device:

#!/usr/bin/env bash
# deck-remote.sh — Cross-device agent bridge via Tailscale
set -euo pipefail
S23_IP="${S23_IP:-100.109.67.85}"
BASE="http://${S23_IP}:${DECK_PORT:-8080}"
TIMEOUT="${DECK_TIMEOUT:-300}"

_esc() { echo "$1" | sed 's/\\/\\\\/g; s/"/\\"/g'; }

case "${1:-help}" in
  ping)
    curl -sf --connect-timeout 5 "${BASE}/api/goals" -o /dev/null \
      && echo "✓ S23 reachable at ${BASE}" || { echo "✗ Cannot reach S23" >&2; exit 1; } ;;
  ask)
    shift; q="$*"
    qid=$(curl -sf -X POST "${BASE}/api/agent/question" \
      -H 'Content-Type: application/json' \
      -d "{\"text\":\"$(_esc "$q")\"}" | python3 -c "import sys,json;print(json.load(sys.stdin)['id'])")
    echo "[deck] Question posted ($qid)" >&2
    elapsed=0; poll=120
    while [ "$elapsed" -lt "$TIMEOUT" ]; do
      t=$(curl -sf "${BASE}/api/agent/response?id=${qid}&wait=${poll}" 2>/dev/null \
        | python3 -c "import sys,json;print(json.load(sys.stdin).get('text',''))" 2>/dev/null || echo "")
      [ -n "$t" ] && { echo "$t"; exit 0; }
      elapsed=$((elapsed + poll))
    done
    echo "Timed out" >&2; exit 1 ;;
  send)
    shift; curl -sf -X POST "${BASE}/api/agent/message" \
      -H 'Content-Type: application/json' -d "{\"text\":\"$(_esc "$*")\"}" -o /dev/null
    echo "[deck] Sent" >&2 ;;
  inbox)
    curl -sf "${BASE}/api/agent/inbox?wait=5" 2>/dev/null \
      | python3 -c "import sys,json;t=json.load(sys.stdin).get('text','');print(t if t else '(empty)')" ;;
  *)
    echo "Usage: deck-remote.sh {ping|ask|send|inbox} [text]"
    echo "  S23_IP=$S23_IP  DECK_PORT=${DECK_PORT:-8080}" ;;
esac

Security Note

The Deck server has no authentication. It's only accessible via Tailscale (private VPN mesh), which provides encryption and access control. Don't expose port 8080 to the public internet.
